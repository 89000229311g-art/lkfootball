# Иерархия Ролей и Прав Доступа

## Структура Ролей

```
┌─────────────────────────────────────────┐
│  🏆 РУКОВОДИТЕЛЬ (super_admin)          │  ← Самая высокая роль
│  Полный контроль над всей системой      │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  🛡️ АДМИНИСТРАТОР (admin)               │  ← Управляет тренерами и родителями
│  Управление пользователями и системой   │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  👨‍🏫 ТРЕНЕР (coach)                      │  ← Работа с группами и учениками
│  Управление группами и посещаемостью    │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  👨‍👩‍👧‍👦 РОДИТЕЛЬ (parent)                │  ← Просмотр информации
│  Просмотр статистики и общение          │
└─────────────────────────────────────────┘
```

## Матрица Прав Доступа

### 1. Управление Паролями

| Текущая роль | Может менять пароль для |
|--------------|------------------------|
| 🏆 Руководитель | ✅ Всем (super_admin, admin, coach, parent) |
| 🛡️ Администратор | ✅ Только coach и parent<br>❌ НЕ может: super_admin, admin |
| 👨‍🏫 Тренер | ❌ Никому |
| 👨‍👩‍👧‍👦 Родитель | ❌ Никому |

**API Endpoint:** `PUT /api/v1/auth/users/{user_id}/password`

**Примеры:**
```bash
# Руководитель меняет пароль администратору
curl -X PUT "http://localhost:8000/api/v1/auth/users/5/password" \
  -H "Authorization: Bearer SUPER_ADMIN_TOKEN" \
  -d "new_password=newpass123"

# Администратор меняет пароль тренеру
curl -X PUT "http://localhost:8000/api/v1/auth/users/8/password" \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -d "new_password=newpass123"
```

## Особенности Иерархии

### Защитные Механизмы

1. **Самозащита:** Никто не может изменить/удалить свой собственный аккаунт через административные эндпоинты
2. **Изоляция уровней:** 
   - Администраторы не могут трогать других администраторов
   - Администраторы не имеют доступа к руководителям
3. **Последний рубеж:** Руководители не могут удалять других руководителей (защита от случайного удаления)

### Важные Правила

✅ **Руководитель (super_admin):**
- Полный контроль над системой
- Может управлять всеми пользователями
- Может назначать и снимать администраторов
- Единственный кто может менять пароли администраторам

✅ **Администратор (admin):**
- Управляет тренерами и родителями
- НЕ может трогать руководителей и других админов
- Второй по важности в иерархии
- Должен обращаться к руководителю для изменений админов

⚠️ **Тренер (coach):**
- Может работать только со своими группами
- Может управлять посещаемостью учеников
- НЕ имеет административных прав
- Может просматривать статистику своих групп

📊 **Родитель (parent):**
- Только просмотр информации о своих детях
- Может писать сообщения
- Может оплачивать через систему (v2.0)
- Самая ограниченная роль

## Примеры Сценариев

### Сценарий 1: Руководитель меняет пароль администратору
```python
# ✅ РАЗРЕШЕНО
current_user: super_admin
target_user: admin
action: change_password
result: SUCCESS
```

### Сценарий 2: Администратор пытается изменить другого администратора
```python
# ❌ ЗАПРЕЩЕНО
current_user: admin
target_user: admin
action: edit_user
result: ERROR "Администратор не может редактировать других Администраторов"
```

### Сценарий 3: Администратор меняет пароль тренеру
```python
# ✅ РАЗРЕШЕНО
current_user: admin
target_user: coach
action: change_password
result: SUCCESS
```

### Сценарий 4: Тренер пытается удалить родителя
```python
# ❌ ЗАПРЕЩЕНО
current_user: coach
target_user: parent
action: delete_user
result: ERROR "У вас нет прав на удаление пользователей"
```

## Коды Ответов API

| Код | Описание |
|-----|----------|
| 200 | Успешно выполнено |
| 400 | Неверные данные (короткий пароль, попытка изменить себя) |
| 403 | Недостаточно прав (нарушение иерархии) |
| 404 | Пользователь не найден |

## Рекомендации по Безопасности

1. **Минимизируйте количество руководителей** - это самая мощная роль
2. **Администраторы для ежедневной работы** - используйте админов для управления
3. **Регулярно проверяйте права доступа** - аудит ролей каждый месяц
4. **Сильные пароли обязательны** - минимум 6 символов (рекомендуется 12+)
5. **Логирование всех изменений** - отслеживайте кто и когда менял пароли
