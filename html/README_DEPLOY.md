# Руководство по деплою Football Academy

Проект готов к развертыванию на сервере с использованием Docker.

## Предварительные требования

На сервере должны быть установлены:
- Docker
- Docker Compose
- Git

## Шаги для деплоя

1. **Клонирование репозитория**
   ```bash
   git clone <repository_url>
   cd football-academy-system
   ```

2. **Настройка переменных окружения**
   Запустите скрипт деплоя, он автоматически создаст файл `.env.production` из примера:
   ```bash
   chmod +x deploy.sh
   ./deploy.sh install
   ```
   **ВАЖНО:** Отредактируйте файл `.env.production` и задайте надежные пароли и секретные ключи!

3. **Сборка фронтенда (если нужно)**
   Фронтенд должен быть собран в папку `frontend/dist`. Если вы деплоите из репозитория, где `dist` не закомичен (что правильно), вам нужно собрать его локально или на сервере (потребуется Node.js):
   ```bash
   # На сервере (если есть Node.js)
   cd frontend
   npm install
   npm run build
   cd ..
   ```
   *Альтернатива:* Собрать локально и загрузить папку `dist` на сервер по SCP/FTP.

4. **Запуск**
   ```bash
   ./deploy.sh start
   ```

5. **Проверка статуса**
   ```bash
   ./deploy.sh status
   ```

## Обслуживание

- **Просмотр логов:** `./deploy.sh logs`
- **Бэкап базы:** `./deploy.sh backup`
- **Обновление:** `./deploy.sh update` (стягивает изменения из git и перезапускает контейнеры)

## Структура Docker

- **backend:** FastAPI приложение (порт 8000, закрыт снаружи)
- **db:** PostgreSQL база данных (данные в volume `postgres_data`)
- **redis:** Кэш и брокер сообщений
- **nginx:** Веб-сервер, раздает статику фронтенда и проксирует API запросы (порты 80/443)
- **scheduler:** Планировщик фоновых задач
- **backup:** Сервис автоматических бэкапов (раз в сутки)

## SSL Сертификат

Для настройки HTTPS отредактируйте `nginx/nginx.prod.conf` и добавьте сертификаты в папку `nginx/ssl`.
