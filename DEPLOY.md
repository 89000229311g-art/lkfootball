# 🚀 Инструкция по деплою проекта Sunny Football Academy

Эта инструкция описывает процесс переноса изменений из локальной среды разработки на Production-сервер.

---

## 🛠 1. Подготовка локально (на вашем компьютере)

Перед отправкой изменений убедитесь, что проект собирается без ошибок.

### Frontend
1. Перейдите в папку фронтенда: `cd html/frontend`
2. Соберите проект: `npm run build`
   - Это создаст папку `dist` с оптимизированными файлами. 
   - Благодаря настройкам Vite, файлы будут иметь уникальные хэши (например, `main.a1b2c3d4.js`), что предотвратит проблемы с кешированием у пользователей.

### Backend
1. Убедитесь, что все новые зависимости добавлены в `html/requirements.txt`.
2. Если вы меняли модели базы данных, создайте миграцию:
   ```bash
   cd html
   # Если вы используете виртуальное окружение:
   source venv/bin/activate
   alembic revision --autogenerate -m "Описание изменений"
   ```

---

## 📦 2. Перенос данных на сервер

### Через GitHub (Исходный код)
Залейте изменения в ваш репозиторий:
```bash
git add .
git commit -m "feat: ваши изменения"
git push origin main
```

### Через FileZilla (Статические файлы и ассеты)
Используйте FileZilla для переноса файлов, которые не хранятся в Git:
1. **Frontend**: Залейте содержимое папки `html/frontend/dist` в аналогичную папку на сервере (обычно используется для раздачи через Nginx).
2. **Uploads**: Если вы добавили новые системные изображения (логотипы и т.д.), залейте их в `html/uploads`.
3. **.env**: Если вы добавили новые переменные окружения, обновите файл `.env` на сервере вручную (НЕ заливайте его через Git).

---

## 🖥 3. Настройка на сервере (через SSH)

После того как код обновлен (через `git pull` на сервере), выполните следующие шаги:

### 1. Обновление зависимостей
```bash
cd html
# Если используете venv:
./venv/bin/pip install -r requirements.txt
```

### 2. Применение миграций базы данных (КРИТИЧНО)
**Никогда не удаляйте базу данных на сервере!** Используйте команду upgrade для безопасного обновления схемы:
```bash
cd html
./venv/bin/alembic upgrade head
```
*Эта команда добавит новые таблицы и поля, сохранив все существующие данные.*

### 3. Перезапуск приложения
Если вы используете Docker:
```bash
docker-compose up -d --build backend
```
Если вы используете PM2 или системный сервис:
```bash
pm2 restart football_backend
# Или
sudo systemctl restart football_academy
```

---

## ⚠️ Важные правила безопасности
1. **Никогда** не выкладывайте файл `.env` в публичный доступ.
2. Всегда проверяйте статус базы данных через `alembic current` перед обновлением.
3. Перед крупными обновлениями делайте бэкап базы данных:
   ```bash
   docker exec football_db pg_dump -U postgres football_academy > backup_$(date +%F).sql
   ```
